# Fuwari 风格迁移审查

## 1. 审查基线

- 上游项目：[`saicaca/fuwari`](https://github.com/saicaca/fuwari)
- 审查提交：`6d39b0dec41282e7852e23e032998a5789abee28`
- 当前实现：`apps/web` 下的 Vue 3 SPA
- 审查原则：迁移视觉语言和阅读体验，不把 Astro/Svelte 静态站点运行机制搬进动态博客。

本轮以 Fuwari 的 `MainGridLayout`、`Navbar`、`PostCard`、`PostMeta`、`Profile`、`WidgetLayout`、文章详情、分页、归档和页脚源码为逐项依据。

## 2. 已迁移的基础规格

| 项目 | Fuwari 基线 | Vue 实现 |
| --- | --- | --- |
| 页面宽度 | `75rem` | `--page-width: 75rem` |
| 桌面侧栏 | `17.5rem` | `--sidebar-width: 17.5rem` |
| 主间距 | `1rem` | 主网格及卡片列表使用 `1rem` |
| 主圆角 | `1rem` | `--radius-large: 1rem` |
| 基准字号 | 移动端 `14px`、桌面端 `16px` | 在 `768px` 断点切换 |
| 主色 | `oklch(0.70 0.14 hue)` | 同一 OKLCH 基线，并允许调整 hue |
| 页面背景 | 浅色 `oklch(0.95 0.01 hue)`、深色 `oklch(0.16 0.014 hue)` | 使用相同数值 |
| 卡片背景 | 浅色白色、深色 `oklch(0.23 0.015 hue)` | 使用相同数值 |
| 正文层级 | 黑/白的 90%、75%、50%、30% 透明度 | 使用相同透明度层级 |
| 按钮状态 | regular、plain、card 三组 OKLCH 状态 | 分别映射 hover 与 active，不再共用近似色 |
| 字体 | Roboto 优先 | Roboto Latin + 中文系统字体回退 |

## 3. 已迁移的组件细节

### 全局布局与导航

- 顶部卡片式导航保持 `4.5rem` 高度、顶部直角和底部圆角；
- 桌面显示完整导航，窄屏切换为菜单面板；
- 提供原版 5 度步进的彩虹主题色滑杆，以及浅色、深色、跟随系统三态循环；
- 桌面悬停主题按钮时显示三种模式的直接选择面板；
- 桌面使用固定宽度侧栏，正文占据剩余空间；
- 移动端顺序为正文、侧栏、页脚，避免个人资料阻断阅读入口；
- 全站启用同一 Banner；桌面首页为 `65vh`，其他公开页面和移动端为 `35vh`；
- 顶部栏按当前 Banner 高度计算阈值，接近正文区域时淡出并向上移；
- 页面滚动超过 `35vh` 后，从正文容器右侧显示桌面端返回顶部按钮。

### 首页文章卡片

- 标题使用粗体、主色竖线和悬停箭头；
- 元信息采用独立图标方块，日期严格使用 `YYYY-MM-DD`，并包含分类和标签；
- 摘要、字数、阅读时间与原版保持同一信息层级；
- 有封面时使用右侧约 `28%` 封面，无封面时显示独立进入按钮；
- 卡片按 50ms 递增延迟载入；
- 移动端文章列表组合为一张整体卡片，文章之间只在左右 `1.5rem` 内使用虚线分隔，封面移到正文上方；
- 卡片标题、正文内边距、无封面进入按钮和 `28%` 封面宽度均使用上游原值。

### 侧栏

- 头像使用方形大图，而非圆形头像；
- 姓名下保留短主色横线；
- 分类、标签组件标题使用主色竖线；
- 分类数量使用独立背景徽标；
- 桌面端组件可吸附，移动端保持原版纵向居中资料卡，而不是另造横向资料样式；
- 分类达到 5 项、标签达到 20 项时按原版折叠至 `7.5rem`，并显示“更多”按钮；
- 多个社交链接使用 `2.5rem` 方形图标按钮。

### 文章详情与辅助页面

- 详情标题改为左对齐信息流，顶部先显示字数和阅读时间；
- 标题保留主色竖线，元信息后使用虚线分隔；
- 上一篇、下一篇为正文卡片外的两张导航卡片；
- 目录在 `1536px` 以上显示，使用层级徽标、滚动渐隐和当前标题状态，窄屏不占用阅读宽度；
- 分页使用两侧箭头、中央数字组和主色当前页；
- 归档使用年份节点、圆点、纵向虚线、日期、标题及标签；
- 关于页使用原版 `px-9 py-6` 的普通 Markdown 内容卡片语义；
- 页脚使用长虚线分隔，并保留动态博客已有的归档、关于和备案入口。

### 本轮取消的近似实现

- 不再使用半透明模糊导航、渐变页面背景或常规卡片阴影；
- 不再使用中文长日期、缩小的移动端文章标题或全宽文章分隔线；
- 不再为标签页、分类页发明大型营销卡片，改为复用 Fuwari 的 WidgetLayout、ButtonTag 和 ButtonLink 视觉；
- 不再在关于页增加原版不存在的标题竖线和装饰横条；
- 详情正文若以一级标题开头，仅在详情展示中移除该首标题，避免与 frontmatter/API 标题重复。

## 4. 有意保留的差异

以下差异不是遗漏，而是前后端分离动态博客与静态主题的职责差异：

| 保留差异 | 原因 |
| --- | --- |
| Vue Router 替代 Astro 路由与 Swup | 当前站点是 Vue SPA，页面数据来自 Go API |
| 远程 API 替代 Astro Content Collections | 文章需要在线发布、编辑、草稿和定时发布 |
| 独立标签页与分类页 | 动态博客需要稳定路由及可管理的分类数据 |
| Artalk 评论 | 评论作为独立服务部署，不复制进博客 API |
| 管理端页面 | Fuwari 是静态主题，不包含内容管理能力 |
| Lucide 图标替代 Material Symbols | 保持本地组件化和按需打包，视觉尺寸按原版校准 |
| 不迁移 Pagefind | 静态索引不适用于当前远程内容模型；搜索留待 Go API 实现 |
| 不迁移 RSS、Sitemap | 应由后端或部署层按公开文章生成，不能使用前端假链接 |
| 不迁移 Astro 图片管线 | 封面和头像使用后端返回的公开资源 URL |
| 使用自有中文内容、站名与头像 | Fuwari 只作为设计基线，不复制其示例站内容 |

## 5. 验证标准

每次调整公共前台后至少执行：

```bash
cd apps/web
npm run type-check
npm test
VITE_USE_MOCK_API=false \
VITE_ARTALK_SERVER=/comments \
VITE_ARTALK_SITE=MyBlog \
npm run build
```

部署服务器还必须把 `/posts/*`、`/tags/*`、`/categories/*` 和 `/admin/*` 等深层路由回退到 `index.html`。Vite 生产预览已验证这些路径均返回 SPA 入口。
